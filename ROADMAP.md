# OWIS Roadmap

This roadmap outlines the strategic milestones and phases of the Open Workspace Intelligence Specification (OWIS) project, from foundation through ecosystem adoption and beyond.

---

## Phase 1 — Foundation
- **Status:** ✅ Complete
- **Goal:** Establish project principles, architecture, and governance.
- **Deliverables:**
  - Constitution
  - Documentation Architecture
  - Core Architecture
  - Reference Architecture

---

## Phase 2 — Specification
- **Status:** ✅ Complete
- **Goal:** Formalize system behavior.
- **Deliverables:**
  - Universal Agent Runtime Specification (UARS)
  - Workspace Intelligence Report Specification (WIR)

---

## Phase 3 — Schema
- **Status:** ✅ Complete
- **Goal:** Formalize structural definitions for OWIS artifacts.

---

## Phase 4 — Reference Runtime
- **Status:** ✅ Complete
- **Goal:** Provide a reference implementation of the OWIS runtime.

---

## Phase 5 — CLI
- **Status:** ✅ Complete
- **Goal:** Deliver a command‑line interface for interacting with OWIS.

---

## Phase 6 — SDK
- **Status:** ✅ Complete
- **Goal:** Expose programmatic APIs for OWIS in JavaScript/TypeScript.

---

## Phase 7 — Registry
- **Status:** ⏳ Planned
- **Goal:** Design and implement a central module registry for OWIS components.

---

## Phase 8 — Certification
- **Status:** ⏳ Planned
- **Goal:** Define validation standards and certification processes for OWIS‑compliant tools.

---

## Phase 12 & 13 — v0.2.0 Planning & Review
- **Status:** ✅ Complete
- **Focus:**
  - RFC Governance and Review
  - Security Review
  - v0.2.0 Scope Lock

---

## Phase 14 — v0.2.0 Implementation Sprint
- **Status:** 🚧 In Progress
- **Completed:**
  - RFC‑003 Schema Linter (`owis lint`)
  - RFC‑001 WIR Graph Model
  - RFC‑002 LLM Native Support (`owis context`)

---

## Phase 15 — v0.2.0 Release Candidate
- **Status:** ✅ Complete
- **Goal:** Publish `v0.2.0‑rc.1` to npm and GitHub for broader testing.

---

## Phase 16 — v0.2.0 Stable Release
- **Status:** ✅ Complete
- **Goal:** Release stable `v0.2.0` after RC validation.

---

## Phase 17 — Ecosystem Bootstrap & Adoption Blueprint
- **Status:** ✅ Ready for Implementation (now executed)
- **Scope:** Documentation‑only ecosystem architecture, adoption strategy, and integration planning.
- **Key Decisions:**
  - Scaffold new semantic documentation structure (`docs/architecture`, `docs/specifications`, `docs/reference`, `docs/guides`, `docs/ecosystem`, `docs/integrations`, `docs/examples`).
  - No migration of existing numbered docs.
  - No changes to VitePress routing.
  - Document public API freeze, registry readiness, and governance.

---

## Phase 18 — Registry Implementation
- **Status:** ⏳ Planned
- **Goal:** Build the actual OWIS module registry, including provider contracts, discovery API, version negotiation, and security hardening.

---

## Phase 19 — Certification Standards
- **Status:** ⏳ Planned
- **Goal:** Establish formal certification criteria, test suites, and compliance tooling for OWIS implementations.

---

## Phase 20 — Adoption & Ecosystem Growth
- **Status:** ⏳ Planned
- **Goal:** Drive community adoption, create plug‑in ecosystems, and publish integration guides for major IDEs and platforms.

---

*Roadmap is living; statuses will be updated as phases progress.*
