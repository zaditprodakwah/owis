# OWIS Labels

This document defines the standard labels used in the OWIS issue tracker. Maintaining a consistent set of labels ensures issues and pull requests are properly categorized, prioritized, and trackable.

| Label | Description | Color |
|-------|-------------|-------|
| `bug` | Something isn't working as expected. | Red |
| `feature` | New feature or request. | Green |
| `documentation` | Improvements or additions to documentation. | Blue |
| `governance` | Changes to project governance, RFCs, or policies. | Purple |
| `security` | Security vulnerabilities or improvements. | Dark Red |
| `performance` | Performance optimization or benchmark related. | Orange |
| `good first issue` | Good for newcomers. | Light Purple |
| `help wanted` | Extra attention is needed from the community. | Light Green |
| `breaking` | Introduces breaking changes. | Yellow |
| `rfc` | Request for Comments. | Teal |
| `question` | Further information is requested. | Pink |
| `discussion` | Open discussion about a potential change. | Grey |
| `duplicate` | This issue or pull request already exists. | Light Grey |
| `invalid` | This doesn't seem right or is out of scope. | Yellow |
| `wontfix` | This will not be worked on. | White |

## Labeling Guidelines
* Always assign at least one category label (e.g., `bug`, `feature`, `documentation`).
* Use `breaking` for anything that violates backward compatibility.
* Prefix internal issues with the appropriate domain if necessary, but keep the core tags clean.
