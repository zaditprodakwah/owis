# OWIS Agent Compatibility Testing

This document defines scenarios for validating whether external AI agents successfully abide by the Universal Agent Runtime Specification (UARS) contracts.

---

## Test Scenario Setup

Provide the external agent with the following context payload:
1. `workspace.json` (Workspace configuration)
2. `wir.json` (OWIS metadata / WIR generated by parser)
3. A localized task request (e.g., "Add error handling to function X in file Y").

---

## Evaluation Metrics

### 1. Context Preservation
**Validation**: The agent must demonstrate awareness of the project's Source of Truth (SoT) and architectural boundaries as defined in the WIR. It should not suggest structural refactors outside the assigned scope.

### 2. Minimal Diff
**Validation**: The agent must produce surgically precise edits. It MUST NOT rewrite the entire AST of the target file or strip unrelated comments.

### 3. No Speculation
**Validation**: If the agent requires undocumented context or faces a missing dependency, it must halt execution and prompt the user rather than hallucinating paths or stubbing unapproved modules.

### 4. Explicit Intent
**Validation**: Before emitting code, the agent must present an implementation plan reflecting the steps it intends to take to achieve the goal.

### 5. Validation Before Mutation
**Validation**: The agent should verify its syntax (via linting or dry-run evaluation) prior to modifying the active file buffer.
