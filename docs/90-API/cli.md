# OWIS CLI API Contract

**Package:** `@prodakwah/owis`  
**Binary:** `owis`  
**Version:** 0.2.0  
**Status:** FROZEN (v0.2.0-rc.1)

## Overview

The OWIS CLI is the primary user-facing tool for scanning workspaces, running lint checks, and generating LLM context payloads.

## Installation

```
npm install -g @prodakwah/owis
```

---

## Global Options

| Flag | Alias | Description |
|------|-------|-------------|
| `--help` | `-h` | Print help screen and exit 0. |
| `--version` | `-v` | Print version string and exit 0. |
| `--output <path>` | `-o` | Override the output path for generated artifacts. |

---

## Commands

### `owis scan <workspace_path>`

Default command. Scans a workspace, validates it, generates `wir.json` and `wir.graph.json`.

**Alias:** `owis <workspace_path>` (implicit scan when no subcommand is given)

#### Arguments

| Argument | Required | Default | Description |
|----------|----------|---------|-------------|
| `workspace_path` | No | `process.cwd()` | Path to the workspace root. |

#### Output Files Written

| File | Location | Description |
|------|----------|-------------|
| `wir.json` | `<workspace_path>/wir.json` | Workspace Intelligence Report. |
| `wir.graph.json` | `<workspace_path>/wir.graph.json` | WIR Graph serialization. |

With `--output <path>`, both files are redirected:
- `wir.json` → `<output>`
- `wir.graph.json` → `<output>` with `.json` replaced by `.graph.json`

#### Stdout Format

```
OWIS Workspace Validation

Workspace:
<absolute-path>

Detected:
✓ Language: JavaScript
✓ Language: Markdown
...
✓ Documentation
✓ Package Metadata (npm)

Generated:
✓ WIR
✓ WIR Graph
  - Nodes: <n>
  - Edges: <n>

Status:
COMPLIANT
```

#### Exit Codes

| Code | Condition |
|------|-----------|
| `0` | WIR valid and files written successfully. |
| `1` | WIR failed schema validation, parse error, or write error. |

---

### `owis lint <workspace_path>`

Runs architecture lint rules against the workspace's WIR.

#### Arguments

| Argument | Required | Default | Description |
|----------|----------|---------|-------------|
| `workspace_path` | No | `process.cwd()` | Path to the workspace root. Must contain `wir.json` or be scannable. |

#### Output Files Written

None. Lint results are printed to stdout only.

> **Note:** `lint.json` is not currently written to disk by the CLI. It is the responsibility of programmatic consumers (SDK) to persist lint results if needed.

#### Stdout Format

Human-readable lint report generated by `formatter.formatCLI()`.

#### Exit Codes

| Code | Condition |
|------|-----------|
| `0` | No `ERROR`-severity diagnostics. |
| `1` | One or more `ERROR`-severity diagnostics found, or lint engine threw. |

---

### `owis context <workspace_path>`

Reads existing OWIS artifacts and generates a sanitized LLM context payload.

#### Arguments

| Argument | Required | Default | Description |
|----------|----------|---------|-------------|
| `workspace_path` | No | `process.cwd()` | Path to the workspace root. |

#### Input Files Read (all optional)

| File | Description |
|------|-------------|
| `workspace.json` | Workspace configuration. |
| `wir.json` | Workspace Intelligence Report. |
| `wir.graph.json` | WIR Graph. |
| `lint.json` | Lint report (if present). |

At least one of the above must exist, or context generation produces a near-empty output.

#### Output Files Written

| File | Location | Description |
|------|----------|-------------|
| `context.json` | `<workspace_path>/context.json` | Sanitized, validated context (JSON). |
| `context.md` | `<workspace_path>/context.md` | Human-readable context (Markdown). |

#### Stdout Format

```
Context generated

Sources:
✓ workspace.json
✓ wir.json
✓ wir.graph.json

Output:
✓ context.json
✓ context.md

Warnings:
<count>

Truncated:
Yes | No
```

#### Exit Codes

| Code | Condition |
|------|-----------|
| `0` | Context generated and validated successfully. |
| `1` | Context failed schema validation or write error. |

## Artifact Schema Compatibility

| Artifact | Schema | Since |
|----------|--------|-------|
| `wir.json` | `docs/20-SCHEMA/wir.schema.json` | v0.1.0 |
| `wir.graph.json` | Graph serialization format v0.2 | v0.2.0 |
| `context.json` | `context/schema.json` | v0.2.0 |
| `context.md` | No formal schema; human-readable only | v0.2.0 |

## Compatibility Guarantee

- Command names (`scan`, `lint`, `context`) are frozen.
- CLI flags (`--help`, `--version`, `--output`) are frozen.
- Exit code semantics are frozen.
- Stdout **format** (prose, emoji, structure) is informational and may change in patch releases.
- Generated **artifact content** is schema-bound and frozen.
