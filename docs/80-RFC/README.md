# Request for Comments (RFC) Governance

This directory contains the Request for Comments (RFC) proposals for OWIS future versions (starting from v0.2.0). 
Before any new feature is implemented, an RFC must be drafted, reviewed, and approved to ensure architectural alignment with OWIS principles.

## RFC Process

1. Copy the `RFC_TEMPLATE.md` to a new file named `RFC-XXX-TOPIC.md`.
2. Draft the proposal detailing the problem statement, motivation, and solution.
3. Every RFC **must** answer:
   - Why is this needed?
   - What problem does it solve?
   - Does it preserve v0.1.0 compatibility?
   - How does it improve OWIS adoption?
4. Seek community feedback and adoption testing input.
5. Implementation can only begin once the RFC is marked as **Approved**.

## Status Definitions
- **Draft**: Initial proposal, gathering feedback.
- **Review**: Under active review by maintainers.
- **Approved**: Accepted for implementation in the specified roadmap phase.
- **Rejected**: Proposal declined.
